### **一、Buffer Pool（缓冲池）**

Buffer Pool是<span style='color:red'>MySQL用来缓存数据页的内存区域，它可以帮助提高数据库的读取性能。</span>当MySQL需要读取数据时，它会首先尝试从Buffer Pool中查找数据，如果数据页已经在Buffer Pool中，则可以直接从内存中读取，而不需要进行磁盘I/O操作，从而提高了读取性能。

Buffer Pool的大小对于数据库的性能影响非常大。如果Buffer Pool设置得太小，可能导致繁的磁盘I/O操作，降低了数据库的读取性能；如果设置得太大，可能会占用过多的内存资源，影响了系统的整体性能。

**<span style='color:red'>chang buffer是普通非唯一索引才有的</span>**， 它减少了随机磁盘访问，所以对更新性能的提升是会很明显的。

```mysql
场景：查询 SELECT * FROM test WHERE id = 1;
假设条件：
* 表 test有 100 万行数据
* id 是主键（聚簇索引）
* 表使用 InnoDB 引擎
* 行大小约 1KB
* Buffer Pool 初始为空
一、假设第一次查询(Buffer Pool 完全空)
读取路径 = 根节点 → 中间节点 → 叶子节点
需要读取页数 = 3页 (16KB × 3 = 48KB)

初始化状态：
Buffer Pool: [空闲页: 8192个, 已用页: 0个]

步骤1：读取根节点页
1. 从Free List获取页框#0
2. 从磁盘读取页#3 (根节点)
3. 放入LRU Old区尾部
状态：Free:8191, LRU:[#0(old)]

步骤2：读取中间节点页
1. 从Free List获取页框#1
2. 从磁盘读取页#256 (中间节点)
3. 放入LRU Old区尾部
状态：Free:8190, LRU:[#0(old), #1(old)]

步骤3：读取叶子节点页
1. 从Free List获取页框#2
2. 从磁盘读取页#1001 (叶子节点，包含id=1的数据)
3. 放入LRU Old区尾部
状态：Free:8189, LRU:[#0(old), #1(old), #2(old)]

步骤4：返回数据
1. 从页框#2提取行数据
2. 更新页#1001的访问时间
3. 1秒后，页#1001会移到Young区

物理读: 3次 (每次16KB = 48KB)
逻辑读: 3次
耗时: ~3ms (假设磁盘10ms/次寻道 + 2ms/16KB传输)

二、后续查询
读取页数 = 1页 (16KB) - 只需要叶子节点
因为根节点和中间节点已经在Buffer Pool中了

查询前状态：
Buffer Pool: Free:8189, LRU:[#0(old), #1(old), #2(young)]

步骤1：查找页#1001
1. 哈希表命中页框#2
2. 页#1001已在Young区
3. 移动到LRU头部

步骤2：返回数据
直接从内存读取，无需磁盘I/O

物理读: 0次
逻辑读: 1次
耗时: ~0.1ms
```

### **二、Flush链表**

Flush链表是用来管理脏页（即已经被修改但尚未写入磁盘的页）的链表结构。

当数据页被修改后，它会被添加到 Flush链表中，表示这些页需要被刷新到磁盘上的数据文件中。

在后台线程或者在适当的时机，InnoDB 存储引擎会将 Flush链表中的脏页刷新到磁盘上，以确保数据的持久性。

### **三、Free链表**

Free链表是用来管理空闲页（即已经被使用但目前没有数据的页）的链表结构。

当数据页不再被使用时（当这些数据页中的某些行被删除或者更新后，这些数据页中的部分空间可能会变为空闲状态。当这些数据页中的空闲空间达到一定的阈值时，InnoDB存储引擎会将这些数据页添加到Free链表中，表示这些页中的空闲空间可以被重新利用。），它会被添加到 Free链表中，表示这些页可以被重新分配给新的数据。

InnoDB 存储引擎会从 Free链表中分配空闲页给新的数据，以减少频繁的磁盘分配操作。

### **四、Lru链表**

LRU算法用于决定哪些数据页应该保留在缓冲池中，以及哪些数据页应该被替换出去。LRU算法通常以链表的形式实现，LRU链表按照数据页最近被访问的时间顺序进行排列，最近被访问的数据页排在链表的前面，而最久未被访问的数据页排在链表的后面。

**<span style='color:red'>当 Buffer Pool满时需要从缓冲池中淘汰数据页，MySQL会根据LRU链表中数据页的顺序，选择最久未被访问的数据页进行淘汰。这样可以保留最近被访问的数据页，提高缓冲池的命中率，从而提高数据库的读取性能。</span>**
